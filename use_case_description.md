### 회원가입 기능

| **Actor Action**                                                             | **System Response**                 |
| ---------------------------------------------------------------------------- | ----------------------------------- |
|                                                                              | 1. 회원가입 화면을 출력한다.        |
| 2. 회원은 ID, 비밀번호, 전화번호를 입력한다. |                                     |
|                                                                              | 3. 회원가입 완료 메시지를 출력한다. |

### 회원 탈퇴 기능

| **Actor Action**                     | **System Response**                                             |
| ------------------------------------ | --------------------------------------------------------------- |
| 1. 회원은 회원 탈퇴 버튼을 클릭한다. |                                                                 |
|                                      | 2. 탈퇴 확인 메시지를 출력한다.                                 |
| 3. 회원은 '확인'을 누른다.           |                                                                 |
|                                      | 4. 모든 이용 권환과 데이터 삭제 후 탈퇴 완료 메시지를 출력한다. |

### 로그인 기능

| **Actor Action**                                                  | **System Response**                       |
| ----------------------------------------------------------------- | ----------------------------------------- |
| 1. 관리자와 회원은 ID와 비밀번호를 입력하고 로그인 버튼을 누른다. |                                           |
|                                                                   | 2. 로그인이 완료되었다는 화면을 출력한다. |

### 로그아웃 기능

| **Actor Action**                             | **System Response**                         |
| -------------------------------------------- | ------------------------------------------- |
| 1. 관리자와 회원은 로그아웃 버튼을 클릭한다. |                                             |
|                                              | 2. 로그아웃 처리 후 시스템 접속을 종료한다. |

### 대여소 등록 기능

| **Actor Action**                                                                                      | **System Response**                    |
| ----------------------------------------------------------------------------------------------------- | -------------------------------------- |
| 1. 관리자는 대여소 이름, 위치(도시, 주소), 보관 수량, 운영 시간 등을 입력하여 대여소 정보를 등록한다. |                                        |
|                                                                                                       | 2. 대여소 등록 완료 메시지를 출력한다. |

### 대여소 조회 기능

| **Actor Action**                                                         | **System Response**                 |
| ------------------------------------------------------------------------ | ----------------------------------- |
| 1. 관리자는 등록된 대여소 리스트를 조회한다.                             |                                     |
|                                                                          | 2. 등록된 대여소 리스트를 출력한다. |
| Extensions                                                               |                                     |
| After step 1. 관리자는 특정 대여소 항목을 선택하여 삭제 버튼을 클릭한다. |                                     |
|                                                                          | 2. 삭제 완료 메시지를 출력한다.     |

### 대여소 상세내용 기능

| **Actor Action**                         | **System Response**                      |
| ---------------------------------------- | ---------------------------------------- |
| 1. 관리자는 특정 대여소 항목을 선택한다. |                                          |
|                                          | 2. 선택한 대여소의 상세 정보를 출력한다. |

### 자전거 등록 기능

| **Actor Action**                                                                                                             | **System Response**             |
| ---------------------------------------------------------------------------------------------------------------------------- | ------------------------------- |
| 1. 관리자는 자전거 ID, 자전거 제품명을 입력하여 자전거를 등록한다. |                                 |
|                                                                                                                              | 2. 등록 완료 메시지를 출력한다. |

### 자전거 대여 기능

| **Actor Action**                         | **System Response**                      |
| ---------------------------------------- | ---------------------------------------- |
| 1. 회원은 특정 자전거를 대여할 수 있다.  |                                          |
|                                          | 2. 대여 완료 메시지를 출력한다. |

### 대여소 검색 기능

| **Actor Action**                         | **System Response**                           |
| ---------------------------------------- | --------------------------------------------- |
| 1. 회원은 이름을 통해 대여소를 검색한다. |                                               |
|                                          | 2. 검색 조건에 맞는 대여소 리스트를 출력한다. |

### 대여소 상세정보 기능

| **Actor Action**                                            | **System Response**                                                                        |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| 1. 회원은 특정 대여소를 선택한다.                           |                                                                                            |
|                                                             | 2. 대여소의 이름, 위치, 사용 가능 자전거 목록 등이 포함된 대여소 상세정보 화면을 출력한다. |
| Extensions                                                  |
| After step 1. 회원은 특정 자전거 즉시대여 버튼을 클릭한다.  |                                                                                            |
|                                                             | 2. 자전거 즉시대여 완료 메시지를 출력하고, 문자 시스템으로 해당 내용을 전송한다.           |
| After step 1. 회원은 특정 자전거 예약 대기 버튼을 클릭한다. |                                                                                            |
|                                                             | 2. 자전거 예약 대기 완료 메시지를 출력하고, 문자 시스템으로 해당 내용을 전송한다.          |

### 자전거 대여 정보 기능

| **Actor Action** | **System Response**                                                                                                         |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------- |
|                  | 1. 회원의 자전거 대여 정보를 출력한다. 리스트엔 자전거 ID, 자전거 제품명을 포함한다. |

### 자전거 반납 기능

| **Actor Action**                                                        | **System Response**                                                                                                          |
| ----------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| 1. 회원은 대여 정보 리스트 중 반납을 원하는 자전거를 선택하고 반납한다. |                                                                                                                              |
|                                                                         | 2. 회원을 외부 결제 서비스로 연결해 사용 시간에 따라 결제하게 하고 해당 자전거를 반납처리한다.                               |
| Extensions                                                              |
| After step 1. 회원은 원하는 경우 식당 예약 버튼을 클릭한다.             |                                                                                                                              |
|                                                                         | After step 2. 시스템은 외부 식당 예약 서비스에 회원을 연결한다.                                                              |
|                                                                         | After step 1. 시스템은 해당 자전거를 예약대기하고 있던 대기 1순위 회원에게 외부 Email 서비스로 예약되었다는 이메일을 보낸다. |

### 예약 대기 조회 기능

| **Actor Action**                                                               | **System Response**                                       |
| ------------------------------------------------------------------------------ | --------------------------------------------------------- |
| 1. 회원은 예약대기 정보 조회를 클릭한다.                                       |                                                           |
|                                                                                | 2. 해당 회원의 예약대기 정보를 출력한다.                  |
| Extensions                                                                     |
| After step 1. 회원은 리스트에서 취소하고자 하는 예약대기 정보 취소를 클릭한다. |                                                           |
|                                                                                | 2. 해당 예약대기가 취소되었음을 알리는 메세지를 출력한다. |

### 요금 조회 기능

| **Actor Action** | **System Response**                       |
| ---------------- | ----------------------------------------- |
|                  | 1. 회원에게 대여 시간 및 요금을 보여준다. |

### 이용 내역 조회 기능

| **Actor Action**                                                          | **System Response**                           |
| ------------------------------------------------------------------------- | --------------------------------------------- |
|                                                                           | 1. 회원의 과거 대여 기록을 날짜별로 보여준다. |
| Extensions                                                                |
| After step 1, 회원은 대여소별로 정렬하여 과거 대여 기록을 조회할 수 있다. |
| After step 1, 회원은 과거 기록 중 특정 항목을 선택하여 삭제할 수 있다.    |

### 자전거 대여 정보 조회

| **Actor Action**                                                                   | **System Response**                                       |
| ---------------------------------------------------------------------------------- | --------------------------------------------------------- |
|                                                                                    | 1. 자전거 대여 정보를 반납 시간 기준 최근순으로 보여준다. |
| Extensions                                                                         |
| After step 1, 관리자는 지역별로 기준을 정렬하여 자전거 대여 정보를 조회할 수 있다. |

### 대여 금액 및 대여 횟수 조회

| **Actor Action**                                                                  | **System Response**                                             |
| --------------------------------------------------------------------------------- | --------------------------------------------------------------- |
|                                                                                   | 1. 대여 금액 및 대여 횟수를 반납 시간 기준 최근순으로 보여준다. |
| Extensions                                                                        |
| After step 1, 관리자는 최근 1주일 단위로 대여 금액 및 대여 횟수를 조회할 수 있다. |
| After step 1, 관리자는 최근 1개월 단위로 대여 금액 및 대여 횟수를 조회할 수 있다. |
| After step 1, 관리자는 최근 1년 단위로 대여 금액 및 대여 횟수를 조회할 수 있다.   |
